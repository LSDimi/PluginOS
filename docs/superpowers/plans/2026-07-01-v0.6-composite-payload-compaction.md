# Compact `validate_ds_compliance` Output — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Shrink the `validate_ds_compliance` response so it is no worse than the five separate audit calls on payload bytes (while keeping its 5→1 round-trip and 5→1 scan win), by dropping the redundant `detail` and `nodeType` fields and adding a one-time `legend`.

**Architecture:** The composite is the only consumer that emits `CheckFinding` fields directly. Project each finding to a compact `{ nodeId, nodeName, check, meta? }` when building `by_severity` (after the existing cap), add a static `legend`, and update the manifest. The shared `CheckFinding` type, the pure check functions, and every standalone op stay untouched.

**Tech Stack:** TypeScript (bridge-plugin, ES2015 + Figma typings), Vitest with a mocked `figma` global, npm workspaces.

In `packages/bridge-plugin/src/operations/validate.ts`, after the `SEVERITY_ORDER` const (line 11), add:

```typescript
interface CompactFinding {
  nodeId: string;
  nodeName: string;
  check: CheckFinding["check"];
  meta?: Record<string, unknown>;
}

/** Project a full CheckFinding to the compact output shape: drop the redundant
 *  human `detail` string and the unconsumed `nodeType`; keep `meta` only when set. */
function toCompactFinding(f: CheckFinding): CompactFinding {
  const c: CompactFinding = { nodeId: f.nodeId, nodeName: f.nodeName, check: f.check };
  if (f.meta !== undefined) c.meta = f.meta;
  return c;
}

const LEGEND: Record<CheckFinding["check"], string> = {
  contrast: "text contrast below WCAG AA",
  style: "raw fill/stroke/text/effect (no style or variable)",
  detached: "frame likely a detached instance",
  spacing: "auto-layout value off the base-unit grid",
  naming: "default layer name",
};
```

- [ ] **Step 4: Compact the `by_severity` build and add `legend` to the result**

In the same file, replace the cap loop and result assembly (lines 107-124, from `// Cap total findings` through the `};` closing the `result` object) with:

```typescript
    // Cap total findings across buckets, P0 first, then project to the compact
    // output shape (drop detail/nodeType; keep meta only when present).
    let budget = MAX_RESULTS;
    const by_severity: Record<Severity, CompactFinding[]> = { P0: [], P1: [], P2: [], P3: [] };
    for (const sev of SEVERITY_ORDER) {
      if (budget <= 0) break;
      by_severity[sev] = buckets[sev].slice(0, budget).map(toCompactFinding);
      budget -= by_severity[sev].length;
    }

    const total = counts.contrast + counts.style + counts.detached + counts.spacing + counts.naming;
    const shown =
      by_severity.P0.length + by_severity.P1.length + by_severity.P2.length + by_severity.P3.length;
    const result = {
      total_nodes: nodes.length,
      counts,
      by_severity,
      legend: LEGEND,
      summary: `Scanned ${nodes.length} nodes. ${total} findings: ${counts.contrast} contrast (P0), ${counts.style} style (P1), ${counts.detached} detached + ${counts.spacing} spacing (P2), ${counts.naming} naming (P3).`,
    };
```
