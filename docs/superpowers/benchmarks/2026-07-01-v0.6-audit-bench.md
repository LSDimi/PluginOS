# v0.6 Audit Benchmark — `validate_ds_compliance` vs. five separate calls

Deterministic mock-figma fixture, **158 nodes**: 156 base (39 per kind — raw
fill / default name / off-grid spacing / failing contrast) plus one component
`INSTANCE` and one detached-look-alike `FRAME` so the `detached` check is
exercised too. This yields ~196 findings, kept just under the composite's shared
200-item `MAX_RESULTS` budget so it does **not** truncate — an apples-to-apples
comparison, not an artifact of dropped findings (the test asserts the composite
emits no truncation `_hint` at this size).

Payload bytes = `JSON.stringify(safeSerialize(result)).length`; approx tokens =
bytes / 4. Wall-clock ms is indicative only (sub-millisecond, not asserted).
Source: [`audit-bench.test.ts`](../../../packages/bridge-plugin/src/operations/__tests__/audit-bench.test.ts).

## Headline: parity on payload, and a clean structural win

| Path | Payload bytes | ~Tokens | Round-trips | Page scans |
|------|--------------:|--------:|:-----------:|:----------:|
| `lint_styles` + `lint_detached` + `lint_naming` + `check_contrast` + `audit_spacing` (5 calls) | 17,141 | 4,285 | **5** | **5** |
| `validate_ds_compliance` (1 call) | 16,947 | 4,237 | **1** | **1** |
| Delta | **−1% (composite smaller)** | — | **−4 round-trips** | **−4 scans** |

Per-op breakdown of the five separate calls: `lint_styles` 7,944 B ·
`lint_detached` 176 B · `lint_naming` 2,306 B · `check_contrast` 3,872 B ·
`audit_spacing` 2,843 B.

## Honest reading

- **Payload is now at parity** — the composite is marginally *below* the five
  separate calls (−1%) on this fixture, no longer larger. This is the result of
  compacting the composite's per-finding shape: findings are now
  `{ nodeId, nodeName, check, meta? }` (the redundant human `detail` string and
  the unconsumed `nodeType` were dropped), with a single top-level `legend`
  mapping each `check` to its meaning instead of repeating a sentence on every
  finding. A ~1% margin on a synthetic fixture is best read as **"no longer a
  payload penalty,"** not as a large payload win.
- **An earlier draft cited a "+62% larger" (and before that a "35% reduction")
  figure.** The +62% was the pre-compaction shape; the "35% reduction" was a
  truncation artifact measured on an oversized 400-node fixture where the
  composite capped at 200 findings while the five separate ops (each with their
  own full 200 budget) did not. Both are superseded by the numbers above.
- **The durable savings are structural:**
  - **Round-trips 5 → 1.** Four MCP request/response envelopes are eliminated.
    Each MCP tool call also carries protocol framing and — more importantly — an
    agent reasoning turn to decide and issue the next call, none of which is
    captured by this payload-only measurement. For a *full* audit this is the
    dominant token cost, and it is where the composite decisively wins.
  - **Page scans 5 → 1.** One `figma.currentPage.findAll()` walk instead of
    five — a real compute/latency saving that grows with page size.

## When to use which

- **Use `validate_ds_compliance`** for a full compliance sweep — one round-trip,
  one scan, a severity-bucketed report, and now no payload penalty. On a large
  page where total findings exceed `MAX_RESULTS` it emits a truncation `_hint`
  and `counts` still reports the true per-category totals.
- **Use a single-purpose op** (`lint_styles`, `audit_spacing`, …) when you only
  need one dimension — its payload is far smaller (e.g. `lint_detached` at
  176 B vs the composite's ~17 KB), so a targeted question stays cheap.

## Caveat

This measures serialized payload only. It does **not** capture MCP round-trip /
envelope overhead or inter-call agent reasoning — the dominant real-world token
cost of issuing five separate tool calls. The composite's true token advantage
for a full audit is therefore **understated** here: on payload it is at parity,
and the 5→1 round-trip / scan reduction is pure upside on top.
